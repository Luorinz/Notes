## 1 本地路径
~/docker/mcsmanager

## 2 卸载残留
```bash
sudo systemctl stop mcsm-web.service 
sudo systemctl stop mcsm-daemon.service
```
## 3 安装步骤
### 3.1 命令行安装
```bash
# 拉取镜像，在中国的服务器需要自己配置加速镜像源
docker pull githubyumao/mcsmanager-daemon:latest
docker pull githubyumao/mcsmanager-web:latest

# 注意：下列命令中，所有的 `${CHANGE_ME_TO_INSTALL_PATH}`
# 需要替换为你实际数据存储的位置，该位置需要被持久化

# 启动 MCSManager 守护进程端
docker run --name mcsmanager-daemon \
-v /etc/timezone:/etc/timezone:ro  \
-v /etc/localtime:/etc/localtime:ro  \
-v ~/docker/mcsmanager/daemon/data:/opt/mcsmanager/daemon/data \
-v ~/docker/mcsmanager/daemon/logs:/opt/mcsmanager/daemon/logs \
-v /var/run/docker.sock:/var/run/docker.sock \
-e MCSM_DOCKER_WORKSPACE_PATH=~/docker/mcsmanager/daemon/data/InstanceData \
-p 24444:24444 \
-d githubyumao/mcsmanager-daemon:latest


# 启动 MCSManager Web 端
docker run --name mcsmanager-web \
-v /etc/localtime:/etc/localtime:ro \
-v ~/docker/mcsmanager/web/data:/opt/mcsmanager/web/data \
-v ~/docker/mcsmanager/web/logs:/opt/mcsmanager/web/logs \
-p 23333:23333 \
-d githubyumao/mcsmanager-web:latest
```

### 3.2 docker compose安装

```yaml
# docker-compose.yml
services:
  web:
    image: githubyumao/mcsmanager-web:latest
    container_name: mcsm-web
    ports:
      - "23333:23333"
    volumes:
      - /etc/localtime:/etc/localtime:ro
      -  ~/docker/mcsmanager/web/data:/opt/mcsmanager/web/data
      - ~/docker/mcsmanager/web/logs:/opt/mcsmanager/web/logs
    networks:
      - mcsm-net

  daemon:
    image: githubyumao/mcsmanager-daemon:latest
    container_name: mcsm-daemon
    restart: unless-stopped
    ports:
      - "24444:24444"
    environment:
      - MCSM_DOCKER_WORKSPACE_PATH=~/docker/mcsmanager/daemon/data/InstanceData
    volumes:
      - /etc/timezone:/etc/timezone:ro
      - /etc/localtime:/etc/localtime:ro
      - ~/docker/mcsmanager/daemon/data:/opt/mcsmanager/daemon/data
      - ~/docker/mcsmanager/daemon/logs:/opt/mcsmanager/daemon/logs
      - /var/run/docker.sock:/var/run/docker.sock
    networks:
      - mcsm-net
networks:
  mcsm-net:
    driver: bridge            
```

执行
`docker compose -f ~/docker/mcsmanager/docker-compose.yaml up -d`
## 4 配置面板

安装并启动之后，你可以通过 `http://<你的公网IP>:23333` 访问面板。

此时你进入面板，应该会出现一些错误，因为面板 Web 端没有成功连接到守护进程端，需要配置让它们联系到一起。

## 5 连接节点

点击顶部导航栏 `节点`，点击右侧的 `新增节点`，填写你的服务器**公网 IP**，密钥和默认的 `24444` 端口。

执行指令 `cat  ~/docker/mcsmanager/daemon/data/Config/global.json` 可以查看守护进程密钥。

具体流程可以参考：[连接其他机器](https://docs.mcsmanager.com/zh_cn/advanced/distributed.html)

远程节点IP地址填写`ws://mcsm-daemon`
高级设置配置localhost:23333 -> localhost: 24444
